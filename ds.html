HTML (HyperText Markup Language)

HTML est le langage de balisage standard utilisé pour structurer et organiser le contenu d’une page Web. Il définit la hiérarchie des titres, des paragraphes, des images et des liens.

Cas d’usage : Sites web, applications web monopages (SPAs), e-mails marketing, documents structurés pour des liseuses.

<!DOCTYPE html>
<html>
  <head><title>Exemple</title></head>
  <body><h1>Bonjour, monde !</h1></body>
</html>

Outils et Services

Éditeurs : Visual Studio Code, Sublime Text, Atom.
Frameworks (basés sur JS) : React, Angular, Vue.js.
Validation : W3C Markup Validation Service.

Ressources

  • MDN Web Docs - HTML
  • HTML Living Standard

XML (eXtensible Markup Language)

XML est un langage conçu pour stocker et transporter des données de manière lisible. Sa flexibilité lui permet de définir des formats de documents personnalisés.

Cas d’usage : Flux RSS, sitemaps pour le SEO, formats de documents (DOCX, ODT), services web SOAP, configuration d'applications (ex: Android).

<note>
  <to>Alice</to>
  <from>Bob</from>
</note>

Outils et Services

Éditeurs spécialisés : Oxygen XML Editor, XMLSpy.
Bibliothèques (Parsers) : lxml (Python), JAXB (Java), System.Xml (.NET).

Ressources

  • W3C XML Specification
  • MDN Web Docs - XML

JSON (JavaScript Object Notation)

JSON est un format léger d’échange de données. Facile à lire et à écrire pour les humains, il est aussi simple à analyser et à générer pour les machines.

Cas d’usage : APIs RESTful, fichiers de configuration (npm, a.k.a `package.json`), communication entre client et serveur dans les applications web modernes.

{
  "nom": "Dupont",
  "age": 30
}

Outils et Services

Validateurs en ligne : JSONLint.
Outils en ligne de commande : jq.
APIs populaires : GitHub API, Twitter API.

Ressources

  • JSON.org (spécification)
  • MDN Web Docs - JSON

Markdown

Markdown est un langage de balisage léger qui permet de formater du texte via une syntaxe simple. Il est conçu pour être facilement convertible en HTML.

Cas d’usage : Fichiers README sur GitHub/GitLab, documentation technique, générateurs de sites statiques (Jekyll, Hugo), applications de prise de notes (Obsidian).

# Titre
**Texte en gras**
[Lien](https://exemple.com)

Outils et Services

Éditeurs : Typora, Obsidian, VS Code (avec prévisualisation).
Convertisseurs : Pandoc, Marked.js.
Plateformes : GitHub, Reddit, Discord.

Ressources

  • Daring Fireball (créateur)
  • CommonMark (standardisation)

LaTeX

LaTeX est un système de composition de documents, idéal pour la production de documents techniques et scientifiques de haute qualité typographique.

Cas d’usage : Articles de recherche, thèses universitaires, livres, présentations (via Beamer), CVs professionnels.

\documentclass{article}
\begin{document}
$E = mc^2$
\end{document}

Outils et Services

Distributions : TeX Live, MiKTeX.
Éditeurs en ligne : Overleaf, Papeeria.
Gestion bibliographique : BibTeX, BibLaTeX.

Ressources

  • The LaTeX Project
  • Overleaf Documentation

CSS (Cascading Style Sheets)

CSS est le langage qui décrit l’apparence visuelle d’un document HTML. Il gère les couleurs, les polices, l'agencement, les animations et le design responsive.

Cas d’usage : Design de sites web, thèmes pour CMS (WordPress), mise en forme de documents XML, création de styles pour l'impression.

body {
  font-family: Arial, sans-serif;
  color: #333;
}

Outils et Services

Frameworks : Bootstrap, Tailwind CSS.
Préprocesseurs : Sass, Less.
Outils de développement : Firefox Developer Tools, Chrome DevTools.

Ressources

  • MDN Web Docs - CSS
  • CSS-Tricks
